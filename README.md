# Personal pi configuration

My global configuration for [pi](https://pi.dev), the coding agent.

## What's included

- `agent/settings.json` — default models, packages, terminal, and UI settings
- `agent/extensions/` — personal TUI extensions and integrations
- `agent/themes/` — custom themes

Machine-specific state is intentionally excluded:

- API credentials (`agent/auth.json`)
- trusted-project decisions (`agent/trust.json`)
- installed packages (`agent/npm/`)
- session transcripts (`agent/sessions/`)

## Install

This repository mirrors `~/.pi`. On a new machine:

```bash
git clone https://github.com/SlothfulDreams/pi-dotfiles.git ~/.pi
```

If `~/.pi` already exists, clone elsewhere and copy the tracked files into it, or back up the existing directory before cloning.

Authenticate pi separately, then start pi normally. Packages listed in `agent/settings.json` may need to be installed on the new machine:

```bash
pi install npm:pi-web-access
pi install npm:pi-vimmode
pi install npm:@ff-labs/pi-fff
```

## Notes

- Extensions run with full process permissions; review changes before enabling them.
- `herdr-agent-state.ts` is managed by Herdr and may be regenerated by that integration.
- The configuration is personal and may contain preferences that are specific to my providers, models, and terminal.
